# Patterns

We have established some recurring implementation patterns that carry a certain
weight in our codebase. Each file in this directory describes a specific
pattern. What problem it solves, how it is solved, and how it gets applied.

We try to establish living patterns that have a canonical implementation to
make implicit concepts explicit. They name the patterns visibly in the codebase,
provide a shared kernel of types and functionality, and allow attaching in code
documentation. Patterns should have matching filenames inside this documentation
directory as well as for their modules in the source code.
